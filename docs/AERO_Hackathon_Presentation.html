# brief-hackatonu.md
Zbudujcie system w 2 dni:
  • Zarządzanie operacjami lotniczymi helikopterami
  • Inspekcje linii przesyłowych 400 kV
  • 4 role użytkowników z RBAC
  • Mapy, trasy KML, workflow akceptacji
  • Audit log, dwujęzyczność (PL/EN)
  • ! Deadline: 48 godzin
Pierwsza myśl: „jak to w ogóle zacząć?"
> Dodaj endpoint POST /api/orders
> Walidacja: crew_weight ≤ helicopter.max_payload
> Dodaj testy pytest dla wszystkich statusów
# Utwórz model FlightOperation
# RBAC: tylko supervisor może zaakceptować
# i18n: przetłumacz cały UI na PL/EN
Magia? Cztery narzędzia.
Zero frameworków „no-code". Każde istnieje od lat. Każde darmowe lub tanie. Każde może odpalić każdy w tej sali.
🤖
Claude Code
CLI agent od Anthropic. My opisujemy co i dlaczego, on pisze kod. Sonnet 4.6 + Opus 4.6 (1M context).
$claude code --plan
🔧
GitHub
Issues = backlog. Branch per feature. PR + CI = quality gate. Zero wyjątków, zero merge bez green.
$gh pr create --fill
📋
GSD workflow
Meta-struktura: milestone → slice → phase → task. Wymusza myślenie zanim każesz agentowi pisać.
$gsd plan-phase
🐳
Docker Compose
5 serwisów, 1 polecenie. Środowisko prezentera = środowisko produkcji. Zero „u mnie działa".
$docker compose up --build

Co zaskoczyło
Sytuacja

Agent napisał tak, jak mu kazaliśmy. To my musieliśmy zauważyć, że to wpadka bezpieczeństwa.
